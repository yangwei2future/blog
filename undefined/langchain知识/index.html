LangChain学习分享

LangChain是什么

LangChain是基于大模型的应用开发框架，是一个开源的Python库，旨在通过以下的方式更轻松地构建基于LLM的应用程序：

  1. 向多种不同的基础模型提供通用接口；
  2. 提供管理Prompt提示的框架，以及提供长期记忆能力、外部数据以及其他代理程序的接口，用于处理LLM无法处理的任务（计算或搜索）

LangChain构成

LangChain由多个模块组成，正如其名称所示，连接不同的模块在一起是LangChain的主要目的。想法是将每个模块链接在一个链中，最终使用该链一次性调用所有模块。

LangChain核心概念

Models I/O

大型语言模型（LLM）是指由具有众多参数的神经网络组成并在大量未标记的文本上进行训练的模型。有许多技术巨头和学术组织都有着自己的LLM，例如：OpenAI的GPT-3/4，Google的LaMDA/PaLM，Meta AI的LLaMA，百度的文心，阿里的千问，讯飞的星火，清华的GLM等等。

LLM模块基本用法

  1. 直接调用
    1. 根据输入的问题，让LLM模块输出答案；
  2. 批量生成
    1. 输入：文本的列表
    2. 输出：文本的列表

Prompt提示

LangChain允许有效地管理、优化和序列化Prompt提示，允许开发者使用模板构建动态提示。它可以根据上下文窗口大小和用作上下文（对话历史记录，搜索结果，以前的答案等）的输入变量适应不同的LLM类型。这有助于从语言模型生成更准确且具有上下文相关性的响应。

  1. 与模型交互的“编程方式”-提示（prompt）
    1. 精确的提示=>理想的结果
    2. 冗余的提示=>稳定的结果
    3. Langchain对提示的封装-提示模版（prompt templates）
  2. 提示模版可以包括：
    1. 对语言模型的指令；
    2. 提供一些简单示例给语言模型从而使模型输出更接近理想结果；
    3. 提给语言模型的问题
    4. 等等
  3. 使用场景
    1. 接收部分参数的提示模版

      提供示例给语言模型从而实现逻辑思维
      少样本学习（few-shot）
      通过少量的样本（案例）让语言模型学会处理某类特定问题。
    2. 需要和微调（fine-tuning）区分
    3. 利用样本选择器筛选样本
      1. 样本数量太多
      2. 不是所有样本都能起到提升输出质量的作用

Index数据连接模块

一些基于大型语言模型的应用经常需要用到模型数据集中没有的数据。针对这个需求，Langchain提供了一系列的工具可以从各种数据源中加载新的数据，转换数据，存储数据以及访问数据。

  1. 文档加载器：从多种不同数据源加载文档；
    1. 输入：各种数据源；
      1. PDF
      2. URL
      3. YouTube
    2. 输出：一系列的Document对象
  2. 文档转换器：拆分文档、丢弃冗余文档等；
    1. 加载文档之后，为了更好适用场景，通常会对其进行转换；简单的例子是希望将长文档拆分成较小的块，避免LLM对提示词文本长度的限制；
    2. Langchain具有许多内置的文档转换器，可以轻松地拆分、合并、过滤文档；
    3. 文本分割器
    4. 分割长文本，根据语义相关性把有关联的文本放在同一个分割段；
    5. 文本分割器工作方式
      1. 将文本拆分为小的、语义有意义的块（通常是句子）
      2. 开始将这些小块组合成较大的块，直到达到一定的体量（初始化分割器时指定的chunk_size）。
      3. 一旦达到该大小，将该块作为独立的文本片段，然后开始创建一个新文本块。为保持块之间的连贯性，一般两个文本块之间有重叠部分。
  3. 文本Embedding模型：将非结构化文本转化为浮点数的列表；
    1. 词嵌入是词语的一种数值化表示方式，一般情况下会将一个词映射到一个高维的向量中（vector）
    2. 不再考虑单词或句子，对象变成高维向量->数字化表达（人理解的语言->机器理解的语言）
  4. 向量数据库：存储和搜索embedding数据；
  5. 检索器：查询向量数据；
    1. 根据非结构化查询语句返回相关联文本的接口。

chain链结构

  1. 概念上，链是在规定的标准下，不同模块的组合。
  2. 链是一个抽象出的概念，并不是一定要链才能完成LLM的调用。

链结构的分类

  1. 基础链结构
    1. LLM chain
    2. Router chain
    3. Sequential chain
    4. Transformation chain
  2. 应用链结构实例
    1. Document chains
    2. Retrieval QA

Memory

历史记录模块

  1. 让LLM能够了解之前进行的对话；
  2. 感觉是Prompt engineering

使用方式

  1. 可用于chain，agent等结构；
  2. 可以对memory进行修改，总结，或自定义操作
  3. 可以利用数据库对历史记录进行存取

Agent

很多时候，人们并不会给出精准的问题或者描述。人的大部分需求都是模糊且复杂的。AI需要系统性的对用户的问题/需求进行分析，从而给出针对复杂问题的答案。

例如：Li Auto公司CEO十年后多少岁？

思维链：

  1. 搜索：谁是Li Auto公司CEO？答：李想
  2. 搜索：理想CEO李想多少岁？答：40
  3. 算数：40+10等于多少？答：50

ReAct：Reasoning and acting（总结归纳并执行）

变种：planning and executing
